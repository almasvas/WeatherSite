# WeatherSite

Веб-приложение на **.NET Framework 4.8 (ASP.NET MVC 5)** для отображения погодных данных.

## Что реализовано

- Один экран с погодой:
  - текущая погода;
  - почасовой прогноз (оставшиеся часы текущего дня + все часы следующего);
  - прогноз на 3 дня.
- Состояния интерфейса:
  - загрузка;
  - ошибка;
  - повторный запрос.
- Локализация описаний погоды через `lang=ru`.
- Геолокация по кнопке:
  - пользователь может разрешить определение текущего местоположения;
  - при отказе/ошибке используется Москва по умолчанию.
- Отображение города в заголовке (`Погода: <город>`), включая русификацию названия через reverse geocoding.

## Источники данных

- WeatherAPI:
  - `current.json`
  - `forecast.json?days=3`
- Дополнительно для человекочитаемого названия города при геолокации:
  - reverse geocoding (Nominatim, `accept-language=ru`).

## Архитектура (кратко)

- `Controllers/WeatherController.cs`
  - endpoint `/Weather/Data` для выдачи агрегированных данных на фронтенд.
- `Services/WeatherApiService.cs`
  - запросы к WeatherAPI;
  - сборка единой `WeatherViewModel`;
  - кэширование и отказоустойчивость.
- `Services/ReverseGeocodingService.cs`
  - определение локализованного названия города по координатам.
- `Contracts/WeatherApiContracts.cs`
  - DTO-контракты внешнего API с `JsonProperty`.
- `Scripts/weather.js`
  - загрузка данных, отрисовка блоков, обработка геолокации.

## Производительность и устойчивость

- Кэш в памяти (`MemoryCache`) для погодных данных:
  - `fresh` TTL: 2 минуты;
  - `stale` TTL: 20 минут.
- Защита от stampede (пер-ключевой lock):
  - при одновременных запросах на один ключ только один запрос идет во внешний API.
- Fallback на stale-данные:
  - если внешний API временно недоступен, отдаются последние валидные данные из кэша.
- Оптимизация payload через `current_fields`, `day_fields`, `hour_fields`.
- Ретраи и таймауты для внешних запросов.

## Как запустить

1. Открыть `WeatherSite.sln` в Visual Studio 2022.
2. Восстановить NuGet-пакеты (если требуется).
3. Запустить проект (`IIS Express`).
4. Открыть главную страницу (`/Weather/Index` по дефолтному роутингу).

## Конфигурация

В `Web.config` (`appSettings`):

- `WeatherApiKey`
- `MoscowLat`
- `MoscowLon`

При необходимости можно заменить API-ключ и дефолтные координаты.

## Ограничения и замечания

- Геолокация в браузере требует разрешения пользователя.
- Название локации зависит от качества reverse geocoding.
- Для production-нагрузки рекомендуется вынести кэш в Redis и добавить централизованные метрики/логи.
